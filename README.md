# Secure-Internal-RAG-Assistant

# Project Overview

This repository contains the architecture and configuration for a secure, serverless Retrieval-Augmented Generation (RAG) system. The solution is designed to provide employees with accurate, cited answers to corporate policy inquiries by grounding a Foundation Model in verified internal data.

# Business Use Case

In an enterprise environment, critical information is often distributed across various documents (PDFs, Wikis, and local drives), leading to operational inefficiencies. Providing employees with a raw Large Language Model (LLM) interface poses significant risks regarding data privacy and hallucinations. This project addresses these issues by:

Restricting the AI's knowledge base to specific, authorised documents.

Enforcing strict data-handling policies through automated guardrails.

Providing verifiable citations for every response to ensure accountability.

The source documents used for this project are mock corporate policies generated by AI to represent standard enterprise data structures, including Remote Work Policies and IT Security Protocols.

# Technical Architecture

The system utilises a serverless architecture to ensure high availability and cost-efficiency.

Ingestion Pipeline: Mock policy documents are stored in Amazon S3. These are processed by Amazon Bedrock Knowledge Bases using Amazon Titan Text Embeddings V2.

Vector Storage: Embeddings are stored and indexed in Amazon OpenSearch Serverless, allowing for high-speed, semantic retrieval of document chunks.

Inference: Amazon Nova Micro is utilised for response generation due to its optimised latency-to-performance ratio for RAG tasks.

Security Layer: All inputs and outputs are filtered through Amazon Bedrock Guardrails.

# Technical Resolution: Service Quota Optimisation

During the implementation phase, the data synchronisation process encountered a 429 ThrottlingException. Investigation revealed that the default service quota for Amazon Titan Text Embeddings V2 on-demand inference requests was set to 0 in the target regions.

Resolution Process:

Fault Analysis: Identified the bottleneck within the AWS Service Quotas console.

Strategic Negotiation: Submitted a formal limit increase request to AWS Support. This involved articulating the technical requirements of the RAG pipeline and the necessity of high-throughput indexing for the embedding model.

Outcome: Successfully secured a quota increase to 6,000 requests per minute (RPM), enabling a seamless and efficient data ingestion process.

# Security and Responsible AI

The implementation prioritises data integrity and security through the following configurations:

PII Redaction: Automated masking of sensitive identifiers, including email addresses and phone numbers, to ensure compliance with privacy standards.

Topic Filtering: Implementation of denied-topic filters to block inquiries regarding executive compensation and sensitive financial data.

Contextual Grounding: Verification that every response is derived directly from the indexed source documents, minimizing the risk of model hallucination.
